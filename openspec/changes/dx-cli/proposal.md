## Why

`cd-extras` is a powerful directory navigation module written entirely in PowerShell, but its single-shell implementation limits its reach and creates performance bottlenecks. Frecency calculations and path expansions block shell execution, and sharing directory history across shells or terminal sessions requires fragile concurrency workarounds (background runspaces, Mutexes over CSV files). A compiled cross-shell binary will eliminate these problems — delivering near-zero latency, robust concurrent state management, and a universal navigation experience across Bash, Zsh, Fish, and PowerShell.

## What Changes

- **New CLI binary** (`dx`): A compiled tool (Rust or Go) that serves as the core engine for path resolution, frecency, session stacks, and bookmarks. It never changes directories itself — it resolves paths and prints them to stdout.
- **Shell hooks** for Bash, Zsh, Fish, and PowerShell: Thin wrapper functions generated by `dx init <shell>` that intercept `cd`, invoke the binary, perform the actual directory change, and report back.
- **Persistent frecency store**: SQLite-backed database at `~/.local/share/dx/` with file-locking for safe concurrent writes from multiple shell sessions.
- **Session-scoped stacks**: Per-PID undo/redo stacks stored in temp files (`/tmp/dx-sessions/<PID>.json`) that auto-clean on reboot.
- **Tab completions**: Dynamic completion generation for paths, frecent entries, and bookmarks via `dx complete`.
- **BREAKING**: The existing pure-PowerShell `cd-extras` module will be deprecated in favor of the binary+hook distribution model once feature parity is reached.

## Capabilities

### New Capabilities

- `path-resolution`: Expanding abbreviated paths (e.g., `p/c/d` → `/projects/code/dir`), traditional traversal (`..`, `~`), step-up navigation, and CD_PATH/fallback directory search.
- `session-stacks`: Per-session (PID-keyed) undo/redo directory history with temp-file storage and automatic cleanup.
- `frecency`: Persistent frecency database (frequency + recency scoring) for `cdf`/`cdr` style jumps, with concurrent-safe writes via SQLite.
- `bookmarks`: Named persistent bookmarks (`mark`/`unmark`) for quick directory aliases.
- `shell-hooks`: Shell-specific hook scripts generated by `dx init <shell>` for Bash, Zsh, Fish, and PowerShell — including auto-cd support via `command_not_found_handler`.
- `completions`: Dynamic tab-completion generation for paths, frecent entries, and bookmarks across all supported shells.

## Impact

- **New binary artifact**: Compiled CLI binary distributed via Cargo/Homebrew, added to user's PATH.
- **New shell integration**: Users add an `eval` line to their shell profile (`.bashrc`, `.zshrc`, `config.fish`, `$PROFILE`).
- **New persistent state**: SQLite database at `~/.local/share/dx/` and session files at `/tmp/dx-sessions/`.
- **Deprecation**: The existing `cd-extras` PowerShell module will be superseded. A migration path is needed for existing users.
- **Dependencies**: Rust or Go toolchain for building; SQLite (embedded); shell-specific APIs for completions.
